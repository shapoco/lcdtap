# SPI コマンドが誤って解釈される

## 現象

SPI スレーブから受信したコマンドが正しく解釈されない。

- SPI から映像を入力しても、初期のカラーバー表示から変化しない。
- 数秒～数十秒に1回、画面(カラーバー)の明暗が反転する。恐らく入力されたコマンドが INVON / INVOFF コマンドと誤って解釈されている。

## 解析状況

- 試しに main.cpp の `g_sl2d->inputCommand(byte)` を `g_sl2d->inputCommand(0x2C)` に書き換えて全てのコマンドを RAMWR として解釈するようにしたところ、色は化けているものの、SPI から入力した像が表示さた。色の化け方は試すたびに変わる。十数回に1回程度、正しい色で表示されるように見えることもある。
    - SPI のバイト単位の同期が取れてない？

## 対応

解析が難航したため、Pico2 の外部で SPI 入力をパラレルに変換する方針にした結果問題は無くなった。
